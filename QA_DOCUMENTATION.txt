# QA Documentation: ITCPH Digital Agri-Booth (3D Website)

## 1. Project Overview
The ITCPH Digital Agri-Booth is an interactive 3D virtual environment designed to provide visitors with information about agricultural training, publications, and tools. It features a gamified experience where visitors can register, participate in trivia, and explore various modules through a 3D interface.

## 2. Technical Stack
- **Frontend Framework:** Vue.js 3 (Composition API)
- **Build Tool:** Vite
- **3D Rendering:** Three.js (GLB models)
- **State Management:** Pinia
- **Routing:** Vue Router
- **Real-time Communication:** Microsoft SignalR
- **Styling:** Vanilla CSS / Scoped Vue Styles
- **Backend Integration:** REST API (apiClient.js)

## 3. Core Features & Functional Modules

### 3.1 3D Environment (Booth View)
- **Interactive Model:** A central 3D Agri-booth model (3DAgri-booth.glb).
- **Hotspots:** Clickable navigation points scattered throughout the 3D scene that trigger specific modules.
- **Camera Controls:** Support for orbiting and zooming to inspect booth details.

### 3.2 Visitor Management (Logbook)
- **Visitor Registration:** Form for new visitors to provide contact info and affiliations.
- **Visitor Login:** Authentication for returning visitors.
- **Item Collection:** Tracking of digital materials collected by the visitor during their session.
- **Feedback System:** Allows visitors to leave feedback on their experience.

### 3.3 Functional Modules (Drawer System)
- **Trivia Module (Bebu Game):** 
    - Multiple-choice questions with difficulty levels.
    - XP (Experience Points) scoring system.
    - Progress tracking (Current vs Total Questions).
- **Calculators Module:** 
    - Directory of agricultural calculators.
    - Detailed instructions and FAQs for each tool.
- **LMS Portal:** 
    - Integration with the Learning Management System.
    - Access to training modules and materials.
- **Content Modules:** 
    - Newsletters, Publications, and IEC Materials browsing.
    - Success Stories and Virtual Tour.
- **Chat Module:** Real-time messaging support.

### 3.4 Admin Dashboard
- **Admin Overview:** High-level analytics of visitor traffic and engagement.
- **Admin Logbook:** Detailed list of all registered visitors and their activities.
- **Admin Feedbacks:** Review and manage visitor comments.
- **User Management:** Control over admin accounts and permissions.

## 5. Edge Case & Error Handling Matrix

| Condition | System Response | QA Verification Step |
|-----------|-----------------|----------------------|
| **Backend Offline** | API Client returns "Connection error" | Verify that red error toasts appear on login/submission. |
| **Interrupted Upload**| Progress bar stops; timeout handled | Disconnect network during material upload; verify UI doesn't freeze. |
| **Invalid GLB Path** | Fallback to 2D Map | Intentionally break GLB path in `boothModel.js`; check for error handling. |
| **Partial API Success**| LMS uses `Promise.allSettled` | Fail only the "Courses" API; verify "My Courses" still loads. |

## 6. Mobile & Cross-Platform Verification
- **Touch Targets:** Ensure all 3D hotspots and drawer close buttons are at least 44x44px.
- **Orientation Change:** Rotating from Portrait to Landscape should resize the 3D canvas without refreshing.
- **Keyboard Overlap:** Ensure the registration form input fields are not obscured by the mobile virtual keyboard.

## 7. Step-by-Step User Journey & Interaction Flow

### 7.1 Landing & Loading
1.  **Visit Website:** User navigates to the root URL.
2.  **Loading Screen:** A preparation overlay appears with a progress bar and ITCPH icon.
3.  **Entry Point:** Once loaded, the user is presented with the 3D Agri-Booth.

### 7.2 The 3D Experience (Navigation)
1.  **Orbit/Zoom:** Use mouse (Left-click drag) to orbit and (Scroll) to zoom.
2.  **Selecting a Hotspot:** Click on a 3D hotspot (e.g., "✦ Bebu Game"). The camera zooms in, and the Module Drawer slides in.

### 7.3 Registration & Login
1.  **Accessing Logbook:** Click a restricted module or the profile area.
2.  **Submit Registration:** Fill out the form and click "Submit Registration". Verify the "Welcome" header appears.

### 7.4 Module Interaction
1.  **Closing:** Click the "×" button or the darkened background.
2.  **Downloading Resources:** Click "Download Now". Verify the `.txt` file download and button state change to "Collected".
3.  **Trivia:** Click an answer (Green/Red feedback) and click "Next Question".

### 7.5 Administrative Path
1.  **Login:** Click "Admin Login" (bottom right) and enter credentials.
2.  **Dashboard:** Navigate between Overview, Logbook, and Feedbacks. Verify data reflects recent visitor actions.
